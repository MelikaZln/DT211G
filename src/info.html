Arbetprocessens beskrivning

Till den här uppgiften har vi en gulp fil som är den viktigaste delen av uppgiften. Den har olika delar som utför det som måste göras enligt uppgiftsbeskrivningen. För de olika delarna behöver man olika paket som importeras från Webbsidan till npm. Så vi börjar med det och efter det gör vi enligt instruktionerna på webbsidan där har vi en box som heter usage och den ger kod till varje paket så man vet vad man ska skriva i gulp-filen. Så man importerar paket från webbsidan och sedan exporterar man en funktion som står i webbsidan.
Med hjälp av gulp-concat som är en Gulp-plugin kan man sammanslå flera filer t ex i den här projektet sammanslåss flera css filer till en, och flera js filer till en. Med hjälp av gulp-terser kan man optimiera och minifiera JS-filer, alltså den tar bort radbrytningar, kommentarer, onödiga tecken etc. Samma sak gör Gulp-cssnano fast för css-filer.
För att kunna optimiera mina bilder använde jag först gulp-imagemin men det funkade inte för mig eftersom den optimerar dessa bildformater: PNG, JPEG, GIF och SVAG och inte avif eller webp. Enligt uppgiftsbeskrivningen måste bildformaten vara avif eller webp. Därför använde jag Gulp-sharp-optimize-images det var lite krångligt att jobba med det eftersom det inte gick att bara kopiera och klistra in usage-boxen från webbsidan jag fick skriva själv hur funktionen skulle jobba.
TIll sist använde jag mig av Gulp-babel som används för att kompilera modern js till äldre versioner. alltså det konverterar js-filerna till det version som behövs på så sätt kan man koda med modernare versioner men på grund av bakåtkompatibilitet kan det användas på all sorts webbläsare.
I Gulp-filen ser vi .pipe() den tar data frpn en tidigare operation och skickar den till nästa operation. Dest betyder destination t ex .pipe(dest('pub')) betyder det att det lagrar data till pub mappen. I delen sökvägar har vi en objekt som lagrar sökvägar till HTML-, CSS- JS-filerna och bilderna.
Watch-task funktionen övervakar ändringar i filerna och automatiskt utför uppgifterna då. med parallel() utför flera uppgifter parallelt alltså samtidigt. med series(...) utförs uppgifterna samtidigt och filändringar bevakas.
I början när jag skrev koderna till gulp så kunde jag inte utföra gulp jag fick detta error: SyntaxError: Cannot use import statement outside a module, jag sökte mycket på nätet och till slut fick jag reda på att jag måste skriva "type": "module" i package.json filen. På så sätt funkade det.
Enligt uppgiftsbeskrivningen har min webbplats tre undersidor (hemsidan-bilder-info). I hemsidan kan man läsa om vad uppgiften handlar om. I bilder har jag fem bilder som jag själv har tagit på mina resor och jag verkligen gillar. I information kan man läsa hur arbetspoccessen har gått till.
Efter att ha gjort projektet använde jag Git för att ladda upp alla filer och varje gång jag ändrade på det gjorde jag också det på git och gjorde commit med lämplig beskrivning.
